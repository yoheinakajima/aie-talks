# Build pipeline — semantic search & vector map

Offline, reproducible scripts that turn `data.js` into the search/discovery
artifacts under `../data/`. Everything runs locally in Node; the embedding
model is bundled in an npm package, so **no model download / no API key** is
required at build time.

## Artifacts produced (in `../data/`)

| File | `window` global | Contents |
| --- | --- | --- |
| `vectors.js` | `AIEWF_VEC` | int8-quantized 384-d embeddings (base64), UMAP 2-D map coords, k-means cluster ids + labels, top-8 nearest neighbors per talk |
| `enriched.js` | `AIEWF_ENRICH` | per-talk LLM keyphrases, named entities, one-line summary |
| `analysis.json` | — | dataset analysis (duplicates, novelty, speakers, schedule density) |

## Model

- **Xenova/all-MiniLM-L6-v2**, q8 ONNX, 384-d, mean-pooled + L2-normalized
  (so cosine similarity == dot product).
- Build-time embedding: [`@plurnk/plurnk-mimetypes-embeddings`](https://www.npmjs.com/package/@plurnk/plurnk-mimetypes-embeddings)
  bundles the weights — runs on `onnxruntime-web`, fully hermetic.
- Front-end query embedding must use the **same model** (e.g. Transformers.js
  `Xenova/all-MiniLM-L6-v2`) so query and corpus share one vector space.

## Steps

```sh
cd scripts && npm install
export SCRATCH=/tmp/aiewf-build && mkdir -p $SCRATCH

node 01-embed.mjs        # SCRATCH=… embed 563 talks -> $SCRATCH/vectors.json  (~60s)
node dump-batches.mjs     # split talks into 12 batches for keyphrase extraction
#   -> keyphrases were generated by Claude agents, one per batch, writing
#      $SCRATCH/keyphrases/batch_NN.out.json  (id, keyphrases, entities, summary)
node 02-reduce.mjs       # UMAP 2-D + k-means (K=20) + nearest neighbors -> reduced.json
node 03-analysis.mjs     # dataset analysis -> analysis.json
OUTDIR=../data node 04-assemble.mjs   # merge + int8-quantize -> ../data/*.js
```

Keyphrase extraction is the one LLM step. It was run as 12 parallel Claude
sub-agents over `$SCRATCH/batches/*.json`; to regenerate, re-issue that step
with any LLM that returns `{id, keyphrases[], entities[], summary}` per talk.

`verify-search.mjs` decodes `data/vectors.js` and runs sample semantic queries
end-to-end as a smoke test.
